## 디버깅 프로세스

```
- [ ] 1. 증상 파악 및 재현 조건 확인
- [ ] 2. 관련 코드/로그/스택트레이스 분석
- [ ] 3. 근본 원인 가설 수립
- [ ] 4. 최소 변경으로 수정
- [ ] 5. 테스트로 수정 검증 + 회귀 체크
```

## 규칙

- 수정 전에 반드시 문제를 재현할 수 있는 조건을 파악한다
- 가설을 명시적으로 서술한다: "X 때문에 Y가 발생한다고 판단"
- 수정은 최소 범위로 한정한다 — 주변 코드 정리 금지
- 수정 전후 diff를 보여준다
- 수정이 다른 기능에 영향을 주지 않는지 확인한다

## 과정 표시
매 단계마다 아래 형식으로 과정을 보여준다:
```
[증상] POST /api/users 에서 500 에러, 로그: "NoneType has no attribute 'id'"
[추적] handler.py:45 → service.py:23 → repo.py:12 에서 None 반환
[가설] DB 쿼리가 결과 없이 None 반환 → null 체크 누락
[검증] repo.py:12 확인 — filter 조건에 deleted_at 미포함
[수정] repo.py:12에 .filter(deleted_at=None) 추가
[확인] 동일 요청 재시도 → 200 OK, 기존 테스트 전체 PASS
```
- 스택트레이스를 따라가며 추적 경로를 보여준다
- 가설을 명시적으로 서술한다
- 수정 전후 diff를 보여준다
