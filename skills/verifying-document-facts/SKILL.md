## 문서 사실확인 프로세스

$ARGUMENTS에 대해:

### 1. 수치/공식/주장 추출

대상 문서를 읽고 검증 가능한 항목을 추출한다:

| 분류 | 예시 | 추출 방법 |
|------|------|----------|
| **수치** | "A100 80GB HBM3", "3.35 TB/s" | 숫자 + 단위 패턴 |
| **공식** | "memory = params × dtype / TP" | 수식 패턴 |
| **주장** | "TP 효율은 NVLink에서 95%" | 성능/효율 관련 서술 |
| **비교** | "A보다 B가 2배 빠르다" | 비교 표현 |

추출 형식:
```
[항목 1] "A100의 메모리 대역폭은 2.0 TB/s" (Line 42)
[항목 2] "KV cache = 2 × layers × kv_heads × head_dim × dtype" (Line 78)
[항목 3] "TP=8에서 통신 오버헤드는 15%" (Line 103)
```

### 2. 코드 내 출처 역추적

각 항목의 출처를 프로젝트 코드에서 역추적한다:

```
[항목 1] "A100 메모리 대역폭 2.0 TB/s"
  → src/engine/llm-dist-sim/constants.ts 또는 gpu-profiles.ts
  → 해당 상수의 출처 주석 확인
  → 일치 여부: ✓ / ✗

[항목 2] "KV cache 공식"
  → src/engine/llm-dist-sim/memory.ts:getKvCacheUsageGb()
  → 실제 구현과 문서 기술 비교
  → 일치 여부: ✓ / ✗
```

### 3. 외부 소스 대조

코드에서 확인된 값을 외부 공식 소스와 대조한다:

**확인 순서:**
1. **GPU 스펙** → NVIDIA 공식 데이터시트
2. **vLLM 동작** → vLLM GitHub 소스코드
3. **수학 공식** → 논문, 교과서
4. **벤치마크 수치** → 공식 벤치마크 결과

**병렬 조사가 필요한 경우:**
- Task tool로 Explore agent를 병렬 투입하여 외부 소스 조사
- 예: vLLM 소스 조사 + NVIDIA 스펙 확인을 동시에

```
[항목 1] A100 메모리 대역폭
  [코드] constants.ts → 2.0 TB/s
  [외부] NVIDIA A100 데이터시트 → HBM2e: 2.0 TB/s ✓
  [판정] 일치

[항목 3] TP=8 통신 오버헤드 15%
  [코드] parallelism.ts → TP_OVERHEAD_PER_GPU: 0.15
  [외부] Megatron-LM 논문 Table 3 → 환경에 따라 5-20%
  [판정] 범위 내이나, 단일 값으로 표현은 [추정]
```

### 5. 자동 기록

## 규칙

1. **코드 우선**: 문서의 수치가 코드와 다르면, 코드가 맞는지 먼저 확인한다.
2. **외부 소스 필수**: 코드에서 확인된 값도 외부 공식 소스와 반드시 대조한다.
3. **추정 표기**: 공식 소스로 확인할 수 없는 값은 [추정]으로 명시한다.
4. **심각도 분류**: 계산에 영향을 미치는 불일치는 HIGH, 표현/단위 차이는 LOW.
5. **병렬 조사**: 여러 외부 소스를 확인해야 할 때 Task tool로 Explore agent를 병렬 투입한다.
