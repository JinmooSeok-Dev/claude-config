## 설명 구조

$ARGUMENTS에 대해:

### 1. 한 줄 요약
- 이것은 본질적으로 무엇인가? (한 문장)

### 2. 비유
- 일상생활에서 찾을 수 있는 가장 가까운 비유
- 비유의 한계도 짧게 언급한다

### 3. 시각화
- ASCII 다이어그램으로 구조/흐름/관계를 보여준다
- 코드라면 실행 흐름을 단계별로 추적한다

### 4. 핵심 메커니즘
- 내부에서 실제로 일어나는 일을 단계적으로 설명
- "왜 이런 설계를 했는가"에 대한 동기

### 5. 흔한 오해
- 이 개념에서 자주 발생하는 실수나 혼동

### 6. 관련 개념
- 함께 알면 좋은 연관 개념 2-3개 (간단히)

## 규칙
- 대화하듯 설명한다
- 복잡한 개념은 작은 단위로 쪼갠다
- 코드 설명 시 파일경로:라인번호를 포함한다

## 과정 표시
설명 과정을 아래 형식으로 보여준다:
```
[질문] "attention은 어떻게 동작해?"
[접근] 먼저 비유로 직관 → 수식으로 형식화 → 코드로 확인
[비유] 도서관에서 질문(Q)과 관련된 책(K)을 찾고, 내용(V)을 가져오는 것
[수식] Attention(Q,K,V) = softmax(QKᵀ/√d)·V — 각 항의 의미 설명
[코드] model.py:128 — MultiHeadAttention.forward() 에서 실제 구현 확인
```
- 어떤 순서로 설명할지 먼저 밝힌다
- 추상→구체, 직관→형식 흐름을 보여준다
