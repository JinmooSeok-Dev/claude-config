## 탐구 프레임워크

$ARGUMENTS에 대해 아래 구조로 탐구한다:

### 1. 지도 그리기 (Big Picture)
- 이 분야가 다루는 핵심 질문은 무엇인가?
- 어떤 하위 분야들이 있는가? (관계도를 ASCII로 그린다)
- 현재 어디까지 알려져 있고, 미해결 문제는 무엇인가?

### 2. 핵심 개념 (First Principles)
- 이 분야를 이해하기 위한 최소 필수 개념 3-5개
- 각 개념을 일상 비유로 설명
- 개념 간의 관계와 의존성

### 3. 수학적/형식적 기초 (해당하는 경우)
- 핵심 수식이나 공리를 단계적으로 유도
- 직관적 의미를 반드시 함께 설명
- "이 수식이 말하는 것은 결국 ..."

### 4. 실제 적용과 연결
- 현실 세계에서의 응용 사례
- 사용자의 기존 지식(ML/AI, 시스템, 인프라)과의 접점
- 실습할 수 있는 구체적인 다음 단계

### 5. 학습 로드맵
- 추천 순서 (선수 지식 -> 핵심 -> 심화)
- 핵심 자료 (논문, 교재, 온라인 강의) 2-3개

## 규칙
- 어려운 개념은 반드시 비유와 시각화를 함께 제공한다
- 수식만 나열하지 않는다 — 직관이 먼저, 형식은 그 다음
- 사용자의 배경(ML/시스템 엔지니어)을 활용해 친숙한 개념과 연결한다
- 깊이와 범위를 사용자에게 확인한다 (입문? 중급? 특정 토픽?)

## 과정 표시
탐구 과정을 아래 형식으로 보여준다:
```
[출발점] 양자 컴퓨팅 → "고전 컴퓨터와 뭐가 다른가?"에서 시작
[핵심 개념] qubit = 0과 1의 중첩 상태 (비유: 동전이 공중에 있는 상태)
[연결] 선형대수의 벡터 |ψ⟩ = α|0⟩ + β|1⟩ — 이미 아는 벡터 표현과 동일
[심화] 측정 시 |α|²확률로 0, |β|²확률로 1 → 확률 분석 스킬과 연결
[현재 위치] 기초 개념 → [여기] → 양자 게이트 → 양자 알고리즘 → 응용
```
- 현재 탐구 위치를 전체 지도에서 표시한다
- 새 개념을 기존 지식과 연결하는 과정을 보여준다
- 깊이를 조절할 때 그 이유를 밝힌다
