## 아이디어 구상 프레임워크

$ARGUMENTS에 대해:

### Phase 1: 발산 (Diverge)
- **현재 상태 파악**: 기존에 무엇이 있고, 어떤 문제/한계가 있는가?
- **자극 질문**:
  - "만약 제약이 없다면?"
  - "반대로 접근하면?"
  - "전혀 다른 분야에서 비슷한 문제를 어떻게 풀었는가?"
  - "사용자의 기술 스택(ML/AI, K8s, 시스템)과 결합하면?"
- **아이디어 목록**: 최소 3-5가지 방향을 제시

### Phase 2: 수렴 (Converge)
각 아이디어에 대해:

| 기준 | 평가 |
|------|------|
| 기술적 실현 가능성 | 높음/보통/낮음 |
| 현재 보유 기술과의 시너지 | 설명 |
| 학습 가치 | 새로 배울 수 있는 것 |
| 임팩트/재미 | 왜 할 가치가 있는가 |
| 필요 리소스 | 예상 시간, 도구, 선수 지식 |

### Phase 3: 구체화 (선택된 아이디어)
- 첫 번째로 해볼 수 있는 최소 실험 (MVP/PoC)
- 필요한 기술 스택
- 예상 구조 (디렉토리, 핵심 컴포넌트)
- 즉시 시작할 수 있는 다음 액션 1가지

## 규칙
- 아이디어를 평가 없이 먼저 충분히 발산한다
- 사용자의 기존 프로젝트와 기술 스택을 적극 활용한다
- 실현 불가능해 보이는 것도 일단 나열한다
- 최종 선택은 사용자에게 맡긴다

## 과정 표시
아이디어 구상 과정을 아래 형식으로 보여준다:
```
[현황] llm-d 프로젝트: 분산 추론은 되지만 동적 스케일링 없음
[발산] ① auto-scaling + KV cache 마이그레이션
       ② speculative decoding 통합
       ③ 멀티 모달리티 라우터
[평가] ①: 실현성 높음 + 기존 K8s 스택 활용 / ②: 학습 가치 높음 / ③: 리소스 큼
[선택 기준] 기술 시너지 × 학습 가치 × 실현 가능성
```
- 발산(아이디어 나열)과 수렴(평가)을 명확히 분리한다
- 평가 기준을 먼저 제시하고 그에 따라 비교한다
