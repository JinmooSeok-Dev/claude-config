## 조사 프로세스

$ARGUMENTS에 대해:

### 1. 조사 범위 확인
- 무엇을 알고 싶은가? (사용자에게 범위 확인)
- 깊이: 개요 수준 / 상세 비교 / 종합 서베이
- 관점: 기술적 / 비즈니스적 / 학술적

### 2. 정보 수집
- 웹 검색으로 최신 정보를 수집한다
- 공식 문서, 기술 블로그, 논문을 우선한다
- 코드베이스 내부 조사가 필요하면 Explore 에이전트를 활용한다

### 3. 정리 및 종합

**개요 형식** (빠른 조사):
- 핵심 요약 (3줄)
- 주요 항목 목록
- 출처 링크

**비교 형식** (도구/기술 비교):
| 항목 | A | B | C |
|------|---|---|---|
| 기능 | | | |
| 성능 | | | |
| 생태계 | | | |
| 적합한 경우 | | | |

**서베이 형식** (종합 조사):
- 배경 및 맥락
- 주요 접근법/학파 분류
- 각 접근법의 핵심 아이디어
- 현재 동향과 미래 방향
- 참고 자료 목록

### 4. 시사점
- 사용자의 맥락(ML/AI, 인프라, 학습)에서 의미하는 것
- 추천 다음 행동

## 규칙
- 출처를 반드시 표시한다 (URL, 논문명, 문서 경로)
- 검증되지 않은 정보는 명시적으로 표시한다
- 최신 정보를 우선한다 (날짜 표기)
- 사용자가 원하는 깊이에 맞춘다 — 과하게 길지 않게

## 과정 표시
조사 과정을 아래 형식으로 보여준다:
```
[질문] vLLM vs TensorRT-LLM 성능 비교 (2026년 기준)
[검색] "vllm tensorrt-llm benchmark 2026" → 3개 소스 확인
[소스 1] vLLM 블로그 (2026.01) — Llama 70B 기준 throughput 비교
[소스 2] NVIDIA 벤치마크 (2025.12) — A100/H100 결과
[소스 3] 커뮤니티 벤치마크 (2026.02) — 실사용 환경
[종합] throughput: TRT-LLM 우세 (10-15%), 유연성: vLLM 우세
[신뢰도] 소스 1,2는 각 벤더 자체 벤치마크 → 편향 가능 [주의]
```
- 어떤 검색어로 어떤 소스를 찾았는지 보여준다
- 소스의 신뢰도와 편향 가능성을 표시한다
- 종합 과정에서 어떤 근거로 결론을 도출했는지 밝힌다
