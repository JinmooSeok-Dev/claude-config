## 워크플로우

체크리스트를 복사하여 진행 상황을 추적하라:

```
- [ ] 0. (새 프로젝트인 경우) README.md 생성
- [ ] 1. 요구사항 분석 및 엣지케이스 파악
- [ ] 2. 실패하는 테스트 작성
- [ ] 3. 테스트를 통과하는 최소 구현
- [ ] 4. 테스트 실행 및 확인
- [ ] 5. 리팩토링 (테스트 유지)
- [ ] 6. 최종 테스트 및 결과 보고
```

## Step 0: 새 프로젝트 초기화

프로젝트 디렉토리에 README.md가 없으면 먼저 생성한다.
글로벌 CLAUDE.md에 정의된 README 템플릿을 따른다.

반드시 포함할 항목:
- 개요, 목표, 요구사항
- **기능 범위 (포함 + 미포함)** — 미포함 항목에 이유도 명시
- 기술 스택, 시작하기

사용자에게 초안을 보여주고 확인 후 다음 단계로 진행한다.

## Step 1: 요구사항 분석

- $ARGUMENTS를 분석하여 구현 범위를 명확히 한다
- 기존 코드를 읽고 프로젝트 컨벤션을 파악한다
- 엣지케이스와 에러 시나리오를 나열한다
- 불명확한 점은 구현 전에 질문한다

## Step 2-5: TDD 사이클

- 한 번에 하나의 테스트만 추가한다
- 테스트 없이 프로덕션 코드를 작성하지 않는다
- 각 단계에서 테스트를 실행하고 결과를 보여준다
- 리팩토링 시 테스트가 깨지지 않는지 확인한다

## Step 6: 완료 보고

구현 결과를 요약한다:
- 추가/변경된 파일 목록
- 테스트 실행 결과
- 알려진 제한사항 (있다면)

## 과정 표시
매 단계마다 아래 형식으로 과정을 보여준다:
```
[분석] 입력 검증 필요 — 빈 문자열, null, 최대 길이 초과 케이스 식별
[설계] validator 함수 분리 → 테스트 용이성 확보
[테스트] test_empty_input: FAIL (아직 미구현)
[구현] validator.py에 validate_input() 추가
[결과] test_empty_input: PASS, test_null: PASS, test_max_length: PASS
```
- 요구사항 분석 → 설계 결정 → 테스트 → 구현 → 결과 순서로 표시
- 테스트 실행 결과를 매번 보여준다
