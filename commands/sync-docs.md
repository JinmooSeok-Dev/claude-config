코드 변경사항을 분석하여 관련 문서(README.md, STATUS.md, 설계 문서 등)의 업데이트가 필요한지 확인하고, 필요하면 업데이트한다.

## 분석 대상

!`git diff --cached --name-only 2>/dev/null || git diff --name-only`

## 프로세스

1. 변경된 파일 목록을 확인한다
2. 프로젝트 내 문서 파일을 탐색한다 (README.md, STATUS.md, docs/, *.md)
3. 각 문서에 대해 변경된 코드와 관련 있는 내용이 있는지 판단한다:
   - 기능 추가/변경 → README의 기능 설명, 사용법 업데이트 필요?
   - API 변경 → API 문서 업데이트 필요?
   - 설정 변경 → 설치/설정 가이드 업데이트 필요?
   - 아키텍처 변경 → 설계 문서 업데이트 필요?
   - 의존성 변경 → 요구사항/설치 방법 업데이트 필요?
4. 업데이트가 필요한 문서와 변경 내용을 제안한다
5. 사용자 확인 후 문서를 수정한다

## 규칙
- 업데이트 불필요하면 "문서 업데이트 불필요" 한 줄로 끝낸다
- 사용자 확인 없이 문서를 수정하지 않는다
