$ARGUMENTS 기능을 체계적으로 구현한다.

## 워크플로우

```
- [ ] 0. (새 프로젝트인 경우) README.md 생성
- [ ] 1. 요구사항 분석 및 엣지케이스 파악
- [ ] 2. 실패하는 테스트 작성
- [ ] 3. 테스트를 통과하는 최소 구현
- [ ] 4. 테스트 실행 및 확인
- [ ] 5. 리팩토링 (테스트 유지)
- [ ] 6. 최종 테스트 및 결과 보고
```

## Step 0: 새 프로젝트 초기화
README.md가 없으면 먼저 생성한다. 포함 항목:
- 개요, 목표, 요구사항
- **기능 범위 (포함 + 미포함)** — 미포함 항목에 이유도 명시
- 기술 스택, 시작하기

사용자에게 초안을 보여주고 확인 후 다음 단계로 진행한다.

## Step 1: 요구사항 분석
- 구현 범위를 명확히 한다
- 기존 코드를 읽고 프로젝트 컨벤션을 파악한다
- 엣지케이스와 에러 시나리오를 나열한다
- 불명확한 점은 구현 전에 질문한다

## Step 2-5: TDD 사이클
- 한 번에 하나의 테스트만 추가한다
- 테스트 없이 프로덕션 코드를 작성하지 않는다
- 각 단계에서 테스트를 실행하고 결과를 보여준다

## Step 6: 완료 보고
- 추가/변경된 파일 목록
- 테스트 실행 결과
- 알려진 제한사항 (있다면)
