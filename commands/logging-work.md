현재 세션에서 수행한 작업을 분석하여 프로젝트 루트의 `WORKLOG.md`에 엔트리를 추가한다.

## 1. 기존 WORKLOG.md 확인
- 파일이 있으면 끝에 추가 (기존 내용 유지)
- 없으면 새로 생성:
  ```markdown
  # Work Log
  작업 단위별 진행 기록.
  ```

## 2. 엔트리 형식

```markdown
---

## [YYYY-MM-DD] 작업 제목
> 작업 유형: 구현 | 버그 수정 | 분석 | 설계 | 학습 | 조사

### 목표
- 이번 작업 단위에서 달성하려 했던 것

### 수행 내용
1. [완료된 세부 작업]

### 발생한 문제와 해결
#### 문제 1: [문제 요약]
- **증상**: 무엇이 잘못되었는가
- **원인 파악 과정**: [시도] → [결과]
- **근본 원인**: 왜 발생했는가
- **해결**: 어떻게 고쳤는가

### 의사결정 기록
| 결정 사항 | 선택한 방안 | 대안 | 선택 이유 |
|-----------|------------|------|-----------|

### 현재 상태
- **완료**: / **미완료**: / **다음 단계**:

### 배운 것 (선택)
```

## 3. 규칙
- 문제 없었으면 "발생한 문제와 해결" 생략
- 의사결정 없었으면 "의사결정 기록" 생략
- 해당하는 섹션만 작성한다
- 엔트리 초안을 먼저 보여주고 사용자 확인 후 기록한다
