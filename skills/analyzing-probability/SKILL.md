## 확률/통계 분석 프레임워크

$ARGUMENTS에 대해:

### 1. 문제 유형 분류

| 유형 | 핵심 질문 | 접근법 |
|------|----------|--------|
| **분포 모델링** | 데이터가 어떤 분포를 따르는가? | fitting, QQ-plot, KS test |
| **추정** | 파라미터/확률의 값은? | MLE, MAP, 베이즈 추정 |
| **가설 검정** | 차이가 유의미한가? | t-test, chi-square, permutation |

| **확률 과정** | 상태가 어떻게 변화하는가? | 마르코프 체인, 포아송 과정 |

### 2. 모델 수립

**변수 정의:**
- 확률 변수 X, Y, ... 가 무엇을 나타내는가?
- 알려진 것 vs 구하려는 것

**가정 명시:**
- 독립성, 정상성, 분포 가정
- 가정이 위반되면 어떤 영향이 있는가?

### 3. 분석 수행

**해석적 풀이 (가능한 경우):**
- 수식을 단계적으로 유도
- 각 단계의 직관적 의미를 함께 설명
- 최종 결과의 "이것이 의미하는 것은 ..."

**수치적 풀이 / 시뮬레이션:**
```python
# 몬테카를로 시뮬레이션 예시
import numpy as np

n_simulations = 100_000
results = np.array([simulate_once() for _ in range(n_simulations)])

mean = results.mean()
ci_95 = np.percentile(results, [2.5, 97.5])
```
- 시뮬레이션 횟수의 근거
- 수렴 확인 방법

### 4. 시스템 성능 확률 분석 (해당하는 경우)

**Tail Latency 분석:**
- p50, p90, p95, p99, p99.9 분포
- heavy tail 여부 확인

**큐잉 모델:**
```
도착률 λ, 처리율 μ, 서버 수 c

이용률 ρ = λ / (c × μ)
평균 대기시간 Wq = ρ / (μ(1-ρ))  [M/M/1]
시스템 내 평균 개체 수 L = λ × W  [Little's Law]
```

**가용성 분석:**
- 단일 컴포넌트: P(failure) = 1 - availability
- 직렬 시스템: P(all up) = ∏ P(up_i)
- 병렬 시스템: P(at least one up) = 1 - ∏ P(down_i)

### 5. 결과 해석

- 수치를 현실 맥락에서 해석한다
- 신뢰 구간/불확실성을 반드시 포함한다
- 가정이 변하면 결과가 어떻게 달라지는가 (민감도)
- 시각화: 분포 그래프, CDF, 히트맵 등

## 과정 표시
매 단계마다 아래 형식으로 과정을 보여준다:
```
[모델] 요청 도착을 Poisson(λ=100/s)으로 모델링
[가정] 요청 간 독립, 처리시간 Exp(μ=120/s) → M/M/1 큐 적용
[풀이] ρ = λ/μ = 100/120 = 0.833
[계산] 평균 대기시간 Wq = ρ/(μ(1-ρ)) = 0.833/(120×0.167) = 41.6ms
[검증] 시뮬레이션(n=100k): Wq ≈ 42.1ms (이론값과 1.2% 차이)
[해석] 이용률 83% → 부하 10% 증가 시 대기시간 급증 위험
```
- 어떤 분포/모델을 선택했는지와 그 이유를 먼저 밝힌다
- 가정을 명시하고 위반 시 영향을 언급한다
- 수식 전개의 각 단계에 직관적 의미를 병기한다
- 가능하면 수치 시뮬레이션으로 해석해를 교차 검증한다
